Por defecto, Vite hace code-splitting automático. Usar `React.lazy` con rutas es el patrón para que cada página sea un chunk separado que se carga solo cuando el usuario la visita.

## Configuración

```tsx
import { lazy, Suspense } from 'react'
import { Routes, Route } from 'react-router-dom'

const Dashboard = lazy(() => import('./pages/Dashboard'))
const Profile = lazy(() => import('./pages/Profile'))
const Settings = lazy(() => import('./pages/Settings'))

export default function App() {
  return (
    <Suspense fallback={<div>Cargando...</div>}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/profile" element={<Profile />} />
        <Route path="/settings" element={<Settings />} />
      </Routes>
    </Suspense>
  )
}
```

## Fallback con skeleton

```tsx
function PageSkeleton() {
  return (
    <div className="animate-pulse p-8">
      <div className="h-8 bg-zinc-200 rounded w-1/3 mb-4" />
      <div className="h-4 bg-zinc-200 rounded w-2/3 mb-2" />
      <div className="h-4 bg-zinc-200 rounded w-1/2" />
    </div>
  )
}

// Usar como fallback del Suspense
<Suspense fallback={<PageSkeleton />}>
```

<Callout variant="info">
  No uses `React.lazy` para componentes pequeños o que se usan en muchas páginas — el overhead de un chunk extra supera el beneficio. Aplícalo a páginas completas.
</Callout>
